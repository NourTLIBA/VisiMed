## **SECURITY ISSUES DETAILED**

### **🔴 CRITICAL**

1. **No Token Expiration**
   - **Risk:** Leaked tokens can be used indefinitely
   - **CVSS Score:** 7.5 (High)
   - **Location:** settings.py
   - **Fix:** Add `REST_FRAMEWORK['DEFAULT_AUTHENTICATION_CLASSES']` token expiration settings

2. **Missing HTTPS Enforcement**
   - **Risk:** Credentials transmitted in plain text on HTTP
   - **CVSS Score:** 8.1 (High)
   - **Location:** settings.py#L50 (DEBUG mode)
   - **Fix:** Add `SECURE_SSL_REDIRECT = True` for production

3. **No Password Reset / Recovery**
   - **Risk:** Locked out users; no account recovery mechanism
   - **CVSS Score:** 6.5 (Medium)
   - **Location:** Missing `/api/auth/password-reset/` endpoint
   - **Fix:** Implement password reset flow with email verification

4. **Demo Credentials Visible**
   - **Risk:** Default accounts can be brute-forced
   - **CVSS Score:** 7.2 (High)
   - **Location:** progress.md
   - **Fix:** Delete demo accounts before production; use environment variables

### **🟡 HIGH**

5. **No Rate Limiting**
   - **Risk:** Brute force attack on login; DoS possible
   - **CVSS Score:** 5.9 (Medium)
   - **Fix:** Add `DEFAULT_THROTTLE_CLASSES` in DRF settings

6. **CORS Open to All Origins**
   - **Risk:** Cross-origin requests not validated
   - **CVSS Score:** 5.3 (Medium)
   - **Location:** settings.py#L108
   - **Fix:** Replace with `CORS_ALLOWED_ORIGINS = ["https://yourdomain.com"]`

7. **Missing Android Permissions**
   - **Risk:** App will crash on first network request
   - **CVSS Score:** 5.0 (Medium)
   - **Fix:** Add `<uses-permission android:name="android.permission.INTERNET" />`
